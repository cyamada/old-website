Learning Python

July 11, 2012

First blog post! I thought I'd throw in a blog section to start posting about smaller projects that I've started working that wouldn't necessarly warrant a super detailed write-up. I'll also have a chance to work on my HTML, which will hopefully allow me to revamp the site at some point.

This first post will be dedicated to my recent work with Python and github, both of which I have just started learning. The Python script I wrote is very simple and doesn't have a lot of safety features built in, which is fine for me as it was just an intro.The syntax of Python is still somewhat foreign to me- in general most of the scripting nuances- as the only extensive coding I've done has been for the Arduino and centered around physical interaction with the environment. Even I know the code is crap. This is just an excuse for me to do something a little more productive that sit around and watch youtube videos.

The goal of this program was to create a parser for Twitter feeds that could identify whether or not a keyphrase, given by the user, was present. It doesn't tell you how many times it found the keyphrase, and I'm pretty sure capitalization matters but it works for its most basic task.

The program is split into three parts: the first determines the URL for the RSS feed of the desired twitter account. The second gathers the keyphrases from either a text file or as inputted by the user. The final searches through the xml data and returns whether or not something has been found.

The only library used in this project is urllib2, which allows for the opening of URLs

The method of appending the Twitter username to the end of the given URL just happens to be an easy way to consistantly generate RSS feeds.

The second part of my code is also fairly straightforward. The user has the option of typing keywords in one at a time until they type 'exit', or by placing a text document with a keyphrase on each line in the same directory as the python script (you'll notice there script does not compensate for spacing or capitalization differences).

Finally, the xml data is read by the search function which returns the first index of a word if it is found, and -1 if it is not.

That's all! Hopefully I'll have more stuff to post soon.
